## 🌊 WAVE A — Design Systems & Component Libraries (7 agents in parallel)

Each agent: read `00-dark-tokens.md` + `08-shadcn-dark.md` first, target ~800-1200 lines, ≥80 `var(--df-*)` references, zero hardcoded hex outside `:root`/Three.js, full a11y guards.

### `references/19-once-ui.md` — Once UI
- **URL:** https://once-ui.com
- **Stack:** React / Next.js + Tailwind
- **Positioning angle:** Full design system with scroll storytelling + cinematic transitions. Fills our "complete site system" gap that shadcn (primitives) and DaisyUI (component classes) don't cover.
- **Key features to document:** Pre-built responsive layout tokens, scroll-based section reveal, dark/light mode, CLI install (like shadcn).
- **DF integration angle:** Map Once UI's token naming to DF vars; document scroll-reveal recipes that use `useReducedMotion` guards.

### `references/20-mantine.md` — Mantine
- **URL:** https://mantine.dev
- **Stack:** React + CSS modules
- **Positioning angle:** 120+ components — biggest library by component count. Massive install base. Built-in notification system, modals, rich inputs, 50+ hooks.
- **Key features to document:** ColorScheme management, custom theme override, hook patterns (`useDisclosure`, `useInputState`, `useDebouncedValue`).
- **DF integration angle:** Map Mantine's `MantineProvider` theme to DF tokens via `theme.colors.df*` slots; show dark-mode override.

### `references/21-heroui.md` — HeroUI
- **URL:** https://heroui.com
- **Stack:** React + Tailwind + React Aria
- **Positioning angle:** Dark-first by default — direct brand match for Darkforge. Lightweight, premium feel, framer-motion built in.
- **Key features to document:** Default dark theme, React Aria accessibility, Framer Motion integration patterns.
- **DF integration angle:** Show how to swap HeroUI's default dark palette with DF neon accents while keeping React Aria's a11y intact.

### `references/22-nyxhora-ui.md` — Nyxhora UI
- **URL:** https://nyxhora.com
- **Stack:** React + Tailwind + shadcn
- **Positioning angle:** Startup-grade for SaaS dashboards, CRMs, admin panels. Shadcn-compatible, rich data-heavy UIs.
- **Key features to document:** Dashboard primitives, table patterns, sidebar patterns, integration with shadcn primitives.
- **DF integration angle:** Show how Nyxhora data tables consume DF tokens; bridge to our existing `08-shadcn-dark.md` patterns.

### `references/23-animata.md` — Animata
- **URL:** https://animata.design
- **Stack:** React + Tailwind + Framer Motion
- **Positioning angle:** Curated open-source animation snippets — motion playground style. Modular, hand-crafted micro-interactions.
- **Key features to document:** 30+ production-ready snippets (hover states, magnetic buttons, marquee, animated cards), drop-in usage.
- **DF integration angle:** Each snippet rewrapped with DF tokens; provide the canonical violet/cyan accent variants.

### `references/24-inspira-ui.md` — Inspira UI
- **URL:** https://inspira-ui.com
- **Stack:** Vue 3 + Nuxt + Tailwind + @vueuse/motion
- **Positioning angle:** Vue ecosystem equivalent of Aceternity. Unique because we're React-first — this opens Darkforge to Vue/Nuxt users. **First Vue reference in the plugin.**
- **Key features to document:** Vue 3 Composition API patterns, `<script setup>`, Nuxt content integration, shadcn-vue compatibility.
- **DF integration angle:** Confirm DF token system works in Vue (CSS vars are framework-agnostic — they do); show `inject` pattern for runtime token resolution.
- **Source-of-truth caveat:** Add prominent note that Vue support is **partial** in v1.1 — most of the plugin still targets React, but inspira-ui specifically opens Vue.

### `references/25-float-ui.md` — Float UI
- **URL:** https://floatui.com
- **Stack:** React, Vue, Svelte, HTML
- **Positioning angle:** Multi-framework — rare in this space. Pre-styled hero/pricing/testimonial sections. Useful for Svelte and plain HTML users.
- **Key features to document:** Per-framework code snippets, no heavy dependencies, copy-paste section templates.
- **DF integration angle:** Show DF token block in plain CSS (works in any framework); per-framework wiring.

## 🌊 WAVE B — Scroll & Storytelling Engines (4 agents in parallel)

### `references/26-lenis-smooth-scroll.md` — Lenis ⭐ HIGHEST PRIORITY
- **URL:** https://lenis.darkroom.engineering
- **Stack:** Framework-agnostic
- **Positioning angle:** Single most-requested smooth scroll library in 2025. Used in nearly every Awwwards-winning site. **Darkforge has zero smooth-scroll coverage today — this is a critical gap.**
- **Key features to document:** Wrapping the whole app with `<Lenis>`, easing functions, GSAP ScrollTrigger pairing, raf lifecycle, virtual scroll handling.
- **DF integration angle:** Lenis is rendering-agnostic — show how to combine with our pattern recipes (hero, scroll-story, 3d-scene) without conflict.
- **Bonus:** Show the canonical "Lenis + GSAP ScrollTrigger" combo recipe — every premium site uses this.

### `references/27-locomotive-scroll.md` — Locomotive Scroll
- **URL:** https://locomotivemtl.github.io/locomotive-scroll
- **Stack:** Framework-agnostic
- **Positioning angle:** Parallax + smooth scroll combo. Agency favorite. Data-attribute driven (no JS config for basic use).
- **Key features to document:** `data-scroll-section`, `data-scroll-speed`, sticky elements, GSAP integration.
- **DF integration angle:** Show parallax patterns with DF glow accents; reduced-motion fallback for users who disable parallax.

### `references/28-scrollreveal.md` — ScrollReveal
- **URL:** https://scrollrevealjs.org
- **Stack:** Framework-agnostic (pure JS)
- **Positioning angle:** Beginner-friendly scroll reveals. Great starting point before jumping to GSAP/Lenis.
- **Key features to document:** `sr.reveal('.hero')` API, configuration options, viewport thresholds.
- **DF integration angle:** Quick "drop scroll reveals into any project" recipe; use cases where AOS / Lenis are overkill.

### `references/29-aos-scroll.md` — AOS (Animate On Scroll)
- **URL:** https://michalsnik.github.io/aos
- **Stack:** Framework-agnostic
- **Positioning angle:** Simplest possible scroll animations — `data-aos="fade-up"` and you're done. Massive install base.
- **Key features to document:** All 20+ built-in animations, per-element config, init options, Next.js integration (init in client component).
- **DF integration angle:** Show how AOS pairs with DF-token-driven hero sections; document SSR-safe patterns for Next.js App Router.

## 🌊 WAVE C — Motion, Loaders & Typography (4 agents in parallel)

### `references/30-motion-universal.md` — Motion (Popmotion)
- **URL:** https://motion.dev
- **Stack:** Framework-agnostic (also React)
- **Positioning angle:** New universal animation library from Framer Motion's creator. Lightweight alternative to GSAP for simple-to-medium animations.
- **Key features to document:** Hardware-accelerated animations, springs, keyframes, sequences, plain JS API vs React API.
- **DF integration angle:** When to choose Motion over Framer Motion (smaller bundle) or over GSAP (no Club GSAP needed).

### `references/31-mojs-bursts.md` — Mo.js
- **URL:** https://mojs.github.io
- **Stack:** Framework-agnostic
- **Positioning angle:** Burst/celebration effects — like-button explosions, confetti, click delights. **Unique niche, no overlap with anything we have.**
- **Key features to document:** Burst API, shape morphing, custom paths, timeline editor.
- **DF integration angle:** Use DF neon accents for burst colors; integrate with Framer Motion for orchestrated celebrations.

### `references/32-moving-letters.md` — Moving Letters
- **URL:** https://tobiasahlin.com/moving-letters
- **Stack:** Framework-agnostic (CSS + anime.js)
- **Positioning angle:** 30+ pre-choreographed typographic animations. Powered by anime.js (which we already cover).
- **Key features to document:** Each animation pattern as a copy-paste recipe, anime.js integration, DF-token-aware versions.
- **DF integration angle:** Cross-reference to our existing `14-animejs.md`; treat as a "snippets layer" on top of anime.js.

### `references/33-ldrs-loaders.md` — LDRS
- **URL:** https://uiball.com/ldrs
- **Stack:** Web Components (works everywhere)
- **Positioning angle:** Production-ready loading components — beyond our skeleton system. Web Component angle is unique (one `<l-ring>` tag and done).
- **Key features to document:** All 40+ loader styles, per-loader config (color, size, speed), framework-agnostic usage, DF color overrides.
- **DF integration angle:** Default to `color="var(--df-neon-violet)"` for every loader; pair with our skeleton system for full loading-state coverage.
